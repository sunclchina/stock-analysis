# 开发团队永久工作原则

> 本文档规定了本系统的核心设计原则、开发规范与交付标准。所有开发团队成员在接到任务时、设计过程中、代码实现后，必须逐条对照检查。

---

## 原则一：启用系统开发专用日记

所有开发活动必须通过 `devlog` 工具记录。每次开发活动记录：
- 执行角色
- 起止时间
- 使用的工具/技能
- 产出文件的路径、大小、修改时间
- 重要的技术决策及理由

开发日志统一存储在 `dev-team/DEVLOG.md`，由 `devlog` 自动维护。

## 原则二：消除模拟数据与硬编码

系统中**禁止**出现：
- 模拟数据（mock data）、假数据、占位数据
- 硬编码的 API 地址、密钥、端口、文件路径
- 写死的配置项、阈值、超时时间

所有数据必须**实时获取**：
- 股票行情 → 通达信本地数据 / 新浪 / 东方财富等数据源
- 配置项 → 环境变量 / 配置文件 / 系统配置模块
- 参数 → 用户输入或运行时动态计算

**例外**：测试用例中可以使用测试数据，但必须标注 `// TEST DATA` 且不被编译到生产代码。

## 原则三：严格按设计文档实现，禁止简化替代

不允许以任何理由：
- 用简化脚本替代设计文档中规定的完整实现
- 跳过文档中指定的模块或功能
- 在实现过程中自行降低功能标准

**遇到文档不清或无法实现的情况**：先通过开发总监（青崖）确认后再动手，不得自行裁减功能。

## 原则四：模块化设计 + 文档对照检查

- 严格遵循设计文档的模块划分
- 每个模块职责单一，模块间通过接口/事件通信
- 设计过程中，工程师必须**随时对照设计文档**检查：
  - 数据流是否正确
  - 接口定义是否匹配
  - 模块边界是否清晰
  - 功能覆盖是否完整

## 原则五：文件可视化与路径规范

所有开发产出必须放在指定目录：
- 架构方案 → `dev-team/architect/`
- 后端代码 → `dev-team/backend-dev/`
- 前端代码 → `dev-team/frontend-dev/`
- 测试用例 → `dev-team/tester/`
- 部署配置 → `dev-team/devops/`

确保文件在正确目录，目录结构清晰可读。开发总监负责验收文件路径合规。

## 原则六：前端通过统一导航菜单展示

前端可视化展示必须通过设计文档中规定的 **"统一导航菜单" Web 页面**呈现，包含：
- 仪表盘、实时行情、智能选股、智能分析、智能预警、系统配置
- 侧栏导航、折叠/展开、主题切换
- 所有功能模块通过导航菜单访问，不得另起门户

前端页面必须实时可见，不允许"纯代码无界面"的交付。

## 原则七：前端展示端口固定 8080

前端开发服务器必须固定在 **8080 端口**。不允许使用其他默认端口（如 3000、5173、8081 等）。相关配置：
- 开发环境：`port=8080`
- 生产环境：`port=8080`
- 如有端口冲突，文档中写明处理策略

---

*以上原则适用于开发六人组全体成员，由开发总监（青崖）监督执行。*
